# Davra — База знаний проекта

> Полная бизнес- и техническая логика проекта. Этот файл — канонический источник,
> чтобы знание не потерялось, даже если чат/доступ пропадут. Обновляется по мере развития.

**Что:** приложение для бронирования чайхан под узбекскую традицию «гап».
**Живой сайт:** https://davra-five.vercel.app · **Бот:** [@Davra_choyxona_bot](https://t.me/Davra_choyxona_bot) · **Репо:** https://github.com/ulugbek3030/davra
**Статус:** MVP в проде (фронт + Telegram-бот). Бэкенд и боевые платежи — следующий этап.

---

## 1. Концепт

В Узбекистане есть традиция **гап (гап-гаштак)** — закрытая группа друзей/родни/одногодок,
которая регулярно (обычно раз в месяц) собирается в **чайхане (choyxona)**. Каждый раз
**ответственный** («чья навбат» — чья очередь) организует застолье: бронирует чайхану,
выбирает **изолированную кабинку**, заранее заказывает основное блюдо (плов, казан-кабоб и т.д.).
У многих гапов есть **касса** — каждый сдаёт фиксированную сумму, по кругу её забирает
дежурный (по сути народный ROSCA).

**Davra оцифровывает этот сценарий:** организатор находит и бронирует чайхану в пару тапов,
заказывает блюдо, вносит задаток и собирает кассу гапа — онлайн, в вебе и Telegram Mini App.

**Ядровой пользователь** — организатор гапа. Ключевая идея: это не разовая бронь, а
повторяющаяся группа → удержание (группа возвращается каждый месяц).

**Имя:** Davra (давра — круг за дастурханом). Альтернативы рассматривались: Gashtak, Navbat.

## 2. Рынок и инсайты (ресёрч)

- **Telegram — канал №1 в UZ:** ~88% охвата, ~18 млн юзеров, мини-аппы популярны → веб-сайт
  и Telegram Mini App из одного кода. Android-first (~78%).
- **CLICK — платёжные рельсы:** 20 млн+ пользователей; доля наличных падает (78%→45%). Свои.
- **Whitespace подтверждён:** resto.uz (~1276 заведений), afisha.uz, top.uz, 2GIS — это
  справочники **без онлайн-брони**. Никто не делает связку «поиск чайханы + предзаказ блюда +
  онлайн-задаток». Yandex/Uzum — это доставка, не бронь.
- **Рынок:** общепит UZ ~$826M (2022), ~29 млн посетителей, ~270к свадеб/год, средний банкет
  $30–50/гость.
- **Урок Tomesto (РФ):** модель «% с полного чека» провоцирует занижение счёта и умирает →
  **берём НЕ % с чека**, а сбор с задатка (deposit-led, как Restoplace).
- **Язык:** Uzbek-Latin (по умолчанию) + Русский + Uzbek-Cyrillic.

## 3. Ключевые продуктовые решения

| Решение | Выбор | Почему |
|---|---|---|
| Стартовый артефакт | Сразу рабочий MVP | Быстро «пощупать» продукт |
| География запуска | **Ташкент** | Плотность спроса, проще онбординг |
| Канал | **Telegram Mini App** + веб (один код) | 88% охвата в UZ |
| Монетизация | **Слоёная (4 модели)** | Диверсификация дохода |
| Платежи | **CLICK Pay + цифровая касса гапа** | Свои рельсы + killer-фича/удержание |
| Языки | uz (latin, default), uz-cyrl, ru | Аудитория читает по-узбекски |

---

## 4. Монетизация (слоёная)

1. **Сервисный сбор на задаток** — % с предоплаты брони (деньги идут через нас → заведению).
   Текущая реализация: задаток = `depositPercent`% от ожидаемого чека + сервис Davra **3%**.
2. **Подписка заведений** — бесплатный листинг + платные тарифы (Start / Pro / Premium):
   продвижение, CRM, аналитика.
3. **Платное продвижение** — буст-пакеты, выше в выдаче.
4. **Финтех / касса гапа** — сбор взносов группы через CLICK (флоат + сервис).

Принцип: **никогда не брать % с полного чека** (урок Tomesto). Депозит проходит через платформу.

## 5. Что построено (MVP)

**Гость (веб + Mini App):**
- Каталог чайхан Ташкента + умные фильтры: район, цена (3 уровня), основное блюдо, удобства
  (Wi-Fi, кондиционер, ТВ, парковка, намазхона, проектор, музыка/тамада, халяль, веранда, детская),
  сортировка. Панель фильтров прячется при скролле вниз (мобайл-UX).
- Карточка заведения: фото-обложка, рейтинг/отзывы, адрес/часы/вместимость, описание, удобства,
  кабинки (типы: кабинка/айван/тапчан/зал), меню, отзывы.
- **Флоу брони** (4 шага): кабинка → дата/время/гости → предзаказ блюда → задаток + сводка.
  Гости автоклампятся под вместимость кабинки. Касса гапа: делёж задатка на участников.
  Оплата через CLICK (сейчас замокана) → экран успеха + «уведомление заведению».
- Регистрация по телефону без OTP (localStorage). В Telegram — авто-логин по имени.

**Заведение (`/owner`):** дашборд (KPI, заполняемость), брони (подтвердить/отклонить),
профиль заведения, меню, кабинки, тарифы + продвижение.

**Админка Davra (`/admin`):** обзор платформы (GMV, выручка по источникам, тренд, топ-заведений),
модерация заведений (одобрить/пауза/отклонить), брони платформы, финансы/выплаты.

**Telegram-бот** (`@Davra_choyxona_bot`): `/start` → запрос контакта → по контакту регистрация +
кнопка «Открыть Davra» (Mini App). Реализован как serverless webhook на Vercel (`/api/telegram`).

**i18n:** 3 языка (uz/uz-cyrl/ru), мгновенное переключение, выбор в localStorage.

## 6. Архитектура и стек

| Слой | Технология |
|---|---|
| Фреймворк | Next.js 16 (App Router) + TypeScript (strict) |
| UI | Tailwind CSS v4 · lucide-react · шрифты Manrope + Lora |
| Данные | **Мок** в `src/lib/*.ts` (venues, ownerData, adminData) — БД ещё нет |
| Бот | Serverless webhook (`src/app/api/telegram/route.ts`) + референс на Python/aiogram (`bot/`) |
| Деплой | Vercel (фронт + serverless), авто-деплой при push в `main` |
| Планируется | FastAPI + PostgreSQL + боевой CLICK Pay |

**Дизайн-система:** тёплая «чайханная» палитра (терракота/clay, шафран, suzani-тил/teal, сливовый),
кремовый фон, мотивы сюзане. Токены — в `src/app/globals.css` (`@theme`).

**Структура кода:**
- `src/app/` — маршруты (главная, `venue/[slug]`, `owner/*`, `admin/*`, `tg-demo`, `api/telegram`).
- `src/components/` — UI (Navbar, Hero, Catalog, VenueCard, BookingSheet, LoginModal, AuthProvider,
  VenueDetailBody/Actions, owner/*, admin/*).
- `src/i18n/` — `messages.ts` (база + enum-переводы) + `guest.ts`/`owner.ts`/`admin.ts` (словари
  по доменам, смержены в `messages`), `LocaleProvider.tsx` (`useT`, `money`, `LanguageSwitcher`).
- `src/lib/` — `venues.ts`, `ownerData.ts`, `adminData.ts`, `utils.ts`, `telegram.ts`.

## 7. i18n

- Локали: **`uz` (латиница, по умолчанию)**, `uz-cyrl`, `ru`. Переключатель UZ/ЎЗ/RU в шапке.
- Клиентский `LocaleProvider` + `useT()` → `t("key")`, `money(n)`, `moneyShort(n)`.
- Enum-переводы (районы, блюда, удобства, типы кабинок, статусы, тарифы) — в `messages.ts → enums`.
- Переведено **всё UI**: главная, бронь, страница заведения, owner, admin, модалка входа.
- По-русски остаётся только **контент-данные** заведений (слоганы, описания, адреса, названия
  блюд) — они станут мультиязычными из БД на этапе бэкенда.

## 8. Авторизация

**Сейчас (фронт-MVP, без бэкенда):**
- Веб: ввод телефона → запись в localStorage (без OTP). Косметическая личность.
- Telegram: авто-логин из `initDataUnsafe` (имя). Бот собирает телефон через «поделиться контактом».

**Обязательно на бэкенде (блок 5):**
- Проверка подписи Telegram `initData` (HMAC-SHA256 с токеном бота) — сервер не должен доверять
  `initDataUnsafe`.
- Реальная серверная авторизация `/owner` и `/admin` (middleware + сессии). Сейчас они `noindex`
  и на мок-данных.
- OTP по SMS для регистрации по телефону.

## 10. Безопасность (аудит + фиксы)

**Закрыто в коде:**
- Webhook **fail-closed** (без `TG_WEBHOOK_SECRET` — отказ; раньше принимал любой POST).
- **Экранирование** имени пользователя перед `parse_mode: HTML` (была HTML-инъекция в ответах бота).
- **Security-заголовки** в `next.config.ts`: CSP `frame-ancestors` (Mini App грузится только в
  Telegram/себе — анти-кликджекинг), `X-Content-Type-Options`, `Referrer-Policy`, `HSTS`,
  `Permissions-Policy`.
- `noindex` для `/owner` и `/admin`.
- Секретов в гите нет; `bot/.env` в gitignore, в трекинге только пустой `bot/.env.example`.

**Действие владельца (ВАЖНО):** токен бота был раскрыт в переписке → **ревокнуть через @BotFather**
(`/revoke`), обновить `BOT_TOKEN` в Vercel и переставить webhook с секретом.

**На бэкенд:** проверка подписи `initData` (HMAC), серверная авторизация owner/admin, серверная
проверка подписи колбэка CLICK (никогда не доверять клиенту об оплате), idempotency платёжных
вебхуков, rate-limiting.

## 11. Нагрузка / масштаб

- **Текущий фронт — статика на CDN Vercel, БД нет** → 100 / 500 / 1000 / 10 000+ онлайн держит
  без проблем (на запрос пользователя сервер не работает). Единственная серверная функция —
  Telegram-вебхук, его дёргает Telegram, не посетители.
- Анти-DDoS Vercel активен (подтверждено: бурст с одного IP получает challenge).
- **Реальные узкие места появятся с бэкендом:** пул соединений к PostgreSQL (#1 — использовать
  Neon/Supabase pooler или pgbouncer), rate-limit CLICK, всплески вебхуков (очередь уведомлений),
  cold starts, N+1 запросы. Кэширование через ISR.
