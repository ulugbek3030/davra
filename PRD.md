# Davra — бронирование чайхан для гапа (Узбекистан)

Платформа, которая помогает «ответственному» организатору гапа найти чайхану,
забронировать изолированную кабинку, заказать основное блюдо заранее, внести
задаток и собрать кассу гапа — в вебе, Telegram Mini App и (позже) мобильном
приложении.

## Кто наш пользователь
- **Организатор гапа** (навбатчи) — ядро. Ему нужно собрать давра в 3 тапа.
- **Гости группы** — платят свою долю в кассу.
- **Заведение** (чайхана/банкетный комплекс) — получает брони и управляет залами.

## Ключевые решения (зафиксировано на старте)
- **Канал №1 — Telegram Mini App.** ~88% охвата, ~18 млн юзеров в UZ. Веб и
  Mini App — один и тот же фронт. Натив-приложение — позже. Android-first (~78%).
- **Старт — Ташкент.** Высокая плотность чайхан и спроса.
- **Язык:** Uzbek-Latin (по умолчанию) + Русский, тоггл Кириллицы.
- **Платежи — CLICK** (мы = CLICK): задаток и касса гапа.
- **Касса гапа — killer-фича:** удержание (группа возвращается каждый месяц) +
  платёжный объём.

## Монетизация (слоёная)
1. **Сервисный сбор на задаток** — ~3% сверху, деньги идут через нас → заведению
   (модель Restoplace: pass-through). **НЕ берём % с полного чека** — кейс Tomesto
   (РФ): рестораны занижают счёт, модель умирает.
2. **Подписка заведений** — бесплатный листинг + платные тарифы (CRM, аналитика).
3. **Платное продвижение** в поисковой выдаче.
4. **Финтех-флоат на кассе гапа** через CLICK.

## MVP v1 — СДЕЛАНО (этот спринт)
- Каталог чайхан Ташкента (9 заведений, seed-данные).
- Фильтры: поиск, район, цена (3 уровня), блюдо, удобства, сортировка.
- Карточка заведения: фото-обложка, рейтинг, удобства, кабинки, меню, отзывы.
- Флоу брони: кабинка → дата/время/гости → предзаказ блюда → оплата.
- Экран оплаты: ожидаемый чек, задаток %, сервис Davra 3%, остаток в чайхане.
- **Касса гапа:** сплит задатка по гостям + «отправить ссылку на оплату».
- Экран успеха + упоминание уведомления заведению и напоминания в Telegram.

## Исследование рынка (кратко)
- Ниша свободна: resto.uz (1276 заведений), afisha.uz, top.uz, 2GIS — справочники
  **без брони**. Никто не делает поиск + предзаказ блюда + задаток.
- Спрос: ~29 млн посетителей общепита, ~270 тыс свадеб/год, чек банкета $30–50/гость.
- Telegram доминирует (~88%), CLICK — лидер платежей (20 млн+), кэш падает 78%→45%.

## Стек
- **Фронт:** Next.js 16 (App Router) + React 19 + TypeScript + Tailwind CSS v4.
- **Иконки:** lucide-react. **Шрифты:** Manrope (текст) + Lora (заголовки).
- **Дизайн:** тёплая «чайханная» палитра, мотивы сюзане. Токены — в `globals.css`.
- **Планируется:** FastAPI + PostgreSQL, CLICK Pay, Telegram Bot API, GCP.

## Запуск
```bash
npm install
npm run dev        # http://localhost:3000
```
В этом окружении dev-сервер поднимается через Claude Preview (`davra`, порт 3210).
